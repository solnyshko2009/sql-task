# Таблицы
Patient (Пациент) — человек, проходящий исследование (пробанд, мать, отец, сиблинг).

Family (Семья) — группа пациентов, связанных родством (обычно трио).

SequencingRun (Запуск секвенирования) — технологическая единица (чей номер запуска, тип секвенирования, путь к файлам).

Analysis (Анализ) — результат биоинформатической обработки для конкретного пациента (пайплайн, референс, покрытие, диагноз).

Project (Проект) — категория исследования (из колонки «Проект»: Новообразования и т.д.).

Diagnosis (Диагноз) — справочник кодов МКБ-10 (Q85.0 и др.).
# Типы связей
Patient ⟶ Family — обязательная (каждый пациент в семье). Кардинальность: многие к одному.

Patient ⟶ ParentChild — необязательная (у пробанда есть родители, у родителей может не быть родителей в БД).

Analysis ⟶ Patient — обязательная (у каждого пациента может быть анализ, но данные показывают, что анализ есть у всех).

Analysis ⟶ SequencingRun — обязательная (каждый анализ привязан к запуску).

Analysis ⟶ Project — необязательная (проект может быть NULL).

Analysis ⟶ Diagnosis — необязательная (диагноз может быть не поставлен).
